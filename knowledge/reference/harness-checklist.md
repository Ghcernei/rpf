# Harness checklist — чек-лист зрелой обвязки

Источник: полевые уроки десяти дней (render-харнес, heartbeat, telemetry,
dry-run, install.sh) — собрано по директиве CEO 09.07.2026
(`decisions/INFO-025`). Обязателен для любого строителя операционной
обвязки (скрипт, демон, телеметрия, рендерер, инсталлятор) — ЛЮБОЙ ролью.

## Правило применения (INFO-025)

Строитель харнеса ОБЯЗАН прогнать чек-лист и записать ответы в run.log —
аналог spawn justification для tools. Пропуск пункта — с записанным
обоснованием, никогда молча.

## Чек-лист (9 пунктов)

1. **State-файл и резюмируемость.** Где живёт состояние? Переживает ли
   смерть процесса? Продолжение с места остановки, а не сначала.
   (Канон: install-state.json, INFO-022.)
2. **Идемпотентность.** Повторный запуск безопасен: на здоровой системе
   — no-op/health-check, на полуживой — докат. (Канон: install.sh
   «проверь→сделай».)
3. **Лимит автопопыток.** Максимум 2, дальше — человеку с КОНКРЕТНЫМ
   действием. Бесконечный retry = зависший харнес, который молчит.
4. **Редакция секретов.** Негативный grep-критерий по всем следам
   (лог, state, телеметрия) пуст. (Канон границы: G-003 — вся дорога
   сам, стоп только у секрета.)
5. **Самодостаточный лог + эстафета.** Лог читается без контекста
   сессии (кто, когда, что, чем кончилось); критические события
   эстафетой уходят в системный журнал проекта (run.log родителя /
   heartbeat.log / delegation.log).
6. **Бюджет-строка.** Что жжёт харнес и сколько (токены, время, вызовы
   API) — одной строкой в следе; невидимая стоимость = неуправляемая.
7. **Зависимости.** Каждая — официальна ли, что делаем при её
   исчезновении, есть ли деградация (работа без неё в урезанном
   режиме). (Урок: launchd/osascript/Bot API — у каждого назван план Б.)
8. **Verify до первого боевого прогона.** Слепая или скриптовая
   проверка ДО того, как харнес коснётся реальных данных/людей.
   (Урок: heartbeat beat 1 — WARN парсера пойман тестовым пульсом.)
9. **Happy-path-вопрос.** Явные ответы на три «что если»: упало на
   середине / запустили дважды / нет сети. Ответ «не может быть» не
   принимается — записывается поведение.

## Прецеденты-иллюстрации (названы CEO)

- **install.sh (resume + health-check)** — канон пунктов 1–2: след как
  орган управления, один механизм на установку/докат/проверку.
- **G-003 (стоп у секрета)** — канон границы пункта 4 и периметра
  ценностей (INFO-012): харнес идёт всю дорогу сам и останавливается
  ровно там, где начинается чужое.
