## Формула

**Слепая зона = «требования миссии» − «покрытие вниманием».**

Не «что не сделано из списка» (это бэклог), а «что миссия требует, о чём
никто не думает» — дефицит внимания, не дефицит исполнения.

## Механика

1. Точка отсчёта — `<project>/qroky/mission.md` (два «зачем», INFO-011).
   Без mission.md аудит невозможен — сначала two-whys intake.
2. Веер линз НАД ПРОЕКТОМ, не над задачей: каждая линза отвечает на один
   вопрос «что миссия требует по моей оси и где в артефактах проекта
   (бэклог, история, код, коммуникации) НЕТ следов внимания к этому?»
3. Линзы — из карты функций бизнеса/домена (по границе этапа, кандидат
   6): продажи, продукт, деньги, риски, люди... — доменный набор из
   пакета вертикали (5 параметров, `vertical-parameters.md`).
4. Синтез по SYNTHESIS-PROTOCOL: конфликты факта разрешаются уликами,
   находки ранжируются ценой слепоты для миссии; результат — панель
   человека (строки/гейты), не отчёт-простыня.
5. Соразмерность (INFO-018): дешёвые линзы S/M; веер над проектом — это
   карта, не экспедиция.
